Overview

Introduction

This manual is designed for GIS practitioners, students, and professionals who want to develop practical skills in QGIS — one of the world's most powerful and widely-used open-source Geographic Information System platforms. The course is structured progressively, starting from data acquisition and working through to professional map production.

By the end of this training, participants will be able to:

  • Obtain, import, and manage geospatial datasets in multiple formats
  • Install and configure plugins to extend QGIS functionality
  • Understand and correctly apply coordinate reference systems (CRS/projections)
  • Apply cartographic styling and symbology to convey meaningful spatial information
  • Perform core spatial analysis operations
  • Edit vector features and attribute data
  • Produce professional print-ready maps using the QGIS Print Layout

📋 Note on Datasets: All datasets used throughout this practical are thematically related to an urban planning and land use scenario, spanning Shapefile (.shp), GeoTIFF, CSV, and Excel (.xlsx) formats.

Section 1

Obtaining Geospatial Data

Before any GIS work can begin, you need data. In this section, we identify and download datasets from trusted open-data sources, all thematically linked to urban and environmental analysis.

1.1 Overview of Data Formats Used

FormatDescription & Use
Shapefile (.shp)Most common vector format. Contains geometry + DBF attribute table. Used for boundaries, roads, and points.
GeoTIFF (.tif)Raster format encoding continuous grid data — elevation, land cover, or satellite imagery.
CSV (.csv)Plain-text table. Used to import tabular location data which QGIS can geocode.
Excel (.xlsx)Spreadsheet used for attribute-rich tabular data joinable to spatial layers.
GeoJSON (.geojson)Lightweight JSON-based vector format, common from web APIs and OpenStreetMap exports.

1.2.5 Population/Facility Data (CSV)

Source: OCHA HDX Portal — Search country + 'health facilities' or 'schools'. Ensure CSV has Latitude/Longitude columns.

1.2.6 Socioeconomic Table (Excel)

Section 3

Importing Data into QGIS

3.1 Importing a Shapefile (Vector)

1
In the Browser Panel, navigate to your 01_Admin_Boundaries/ folder
2
Drag the .shp file onto the Map Canvas, OR go to Layer > Add Layer > Add Vector Layer (Ctrl+Shift+V)
3
Click Add, then Close

💡 Tip: A shapefile is a collection of files (.shp, .dbf, .shx, .prj, etc.) that must all stay in the same folder. When copying, always copy ALL files with the same base name.

3.2 Importing a GeoTIFF (Raster)

1
Layer > Add Layer > Add Raster Layer (Ctrl+Shift+R)
2
Navigate to your 03_Raster/ folder and select the GeoTIFF file
3
Drag the raster below vector layers in the Layers Panel

3.3 Importing a CSV File as Point Layer

1
Layer > Add Layer > Add Delimited Text Layer (Ctrl+Shift+T)
2
Browse to your CSV. Under Geometry Definition, select Point coordinates
3
Set X field = Longitude, Y field = Latitude. CRS = EPSG:4326
4
Click Add — points appear on the canvas

3.4 Managing Layers

  • Reordering: Drag layers in the Layers Panel — rasters at bottom, points at top
  • Renaming: Right-click layer → Rename Layer
  • Removing: Right-click → Remove Layer (does not delete the file)
  • Groups: Right-click Layers Panel → Add Group; drag related layers in
  • Properties: Double-click any layer → symbology, labels, fields, joins

1

3
Ensure basemap is at the BOTTOM of the Layers Panel

💡 Basemap CRS: Most basemaps use Web Mercator (EPSG:3857). QGIS reprojects other layers on-the-fly.

Section 5

Coordinate Reference Systems & Projections

Key Concepts

  • Geographic CRS: Angular units (degrees lat/lon). Global data. Example: WGS 84 (EPSG:4326)
  • Projected CRS: Linear units (metres). Local analysis. Example: UTM zones, national grids
  • On-the-fly reprojection: QGIS displays layers in different CRS by reprojecting to project CRS — does NOT change the file's CRS

Setting the Project CRS

1
Project > Properties > CRS tab
2
Filter WGS 84 → select EPSG:4326, OR filter UTM → select zone (e.g. EPSG:32637 for East Africa)
3
Verify CRS in the Status Bar at bottom-right

Reprojecting a Layer (Permanently)

1
Right-click layer → Export > Save Features As
2
Click the CRS selector → choose target CRS (e.g. EPSG:32637)
3
Set output file path → click OK

✅ Best Practice: Always reproject to a projected CRS (UTM) before buffer, area, or distance analysis. Using degrees produces incorrect results.

Section 6

Styling and Symbology

6.1 Single Symbol

Double-click layer → Properties > Symbology → click symbol preview → change Fill, Stroke, Size → OK

6.2 Categorised Renderer (Qualitative)

Use for categorical attributes (e.g. road type, land use class).

1
Symbology → change dropdown to Categorized
2
Set Value to the field (e.g. fclass)
3
Click Classify → auto-generates colours → OK

6.3 Graduated Renderer (Quantitative)

Use for continuous numeric data (population density, elevation).

1
Symbology → Graduated → set Value field
2
Choose Colour Ramp (e.g. YlOrRd for population)
3
Set Mode: Natural Breaks (Jenks) — 5 classes standard → Classify → OK

6.4 Raster Symbology

DEM: Render type = Singleband pseudocolor → choose Colour ramp (Spectral reversed) → Classify → OK
For hillshade: Render type = Hillshade → set sun angle and azimuth

6.5 Adding Labels

1
Properties > Labels → change to Single labels
2
Set Value to name field (e.g. NAME_1)
3
Under Buffer sub-tab → enable Draw text buffer (white) for readability
Section 7

Spatial Analysis

Access via Processing > Toolbox (Ctrl+Alt+T)

7.1 Select by Attribute

Vector > Research Tools > Select by Expression → type query e.g. "population" > 50000 → Select Features

7.2 Buffer Analysis

Creates zones of specified distance around features (e.g. 500m around health facilities).

1
Processing Toolbox → search Buffer → Vector geometry > Buffer
2
Set Input Layer, Distance (e.g. 1000 for 1km — layer must be in metres)
3
Segments = 30 (smooth circle) → Run

7.3 Clip

Vector > Geoprocessing Tools > Clip → Input = layer to clip, Overlay = clipping boundary → Run

7.4 Dissolve

Vector > Geoprocessing Tools > Dissolve → set Dissolve Field (blank = merge ALL into one) → Run

7.5 Spatial Join

Processing Toolbox → Join attributes by location → Input = receiving layer, Join = providing layer → predicate: within → Run

7.6 Zonal Statistics

Processing Toolbox → Zonal statistics → Raster = DEM, Vector = boundaries → select Mean/Max/Min/Sum → Run (stats added as new columns)

7.7 Table Join (Excel to Shapefile)

1
Double-click shapefile → Properties > Joins → click +
2
Set Join Layer = Excel table, Join Field = matching ID column, Target Field = shapefile ID column
3
OK → open Attribute Table — Excel columns are now appended
Section 8

Editing Vector Data

8.1 Enabling Edit Mode

Select layer → click yellow pencil icon in Digitizing Toolbar, OR Layer > Toggle Editing. Save regularly: Ctrl+S

8.2 Editing Attributes

Enable Edit → open Attribute Table (F6) → click a cell to edit → save

8.3 Adding New Features (Digitising)

1
Enable Edit → click Add Point/Line/Polygon Feature in Digitizing Toolbar
2
Click canvas to place vertices. Right-click to finish. Fill attribute form
3
Save edits

8.4 Field Calculator

Attribute Table → click abacus icon → check Create a new field → enter expression e.g. $area / 1000000 (m² to km²) → OK

Section 10

Additional Topics

10.1 Georeferencing a Scanned Map

1
Layer > Georeferencer → load scanned image
2
Add at least 4 Ground Control Points (GCPs) spread across the image
3
Set Transformation type, Resampling method, Target CRS → Start Georeferencing

10.2 Creating a Heatmap

Load point layer → Layer > Properties > Symbology → change renderer to Heatmap → adjust Radius and Colour Ramp

For raster heatmap: Processing Toolbox > Heatmap (Kernel Density Estimation)

10.3 Generating Hillshade from DEM

Processing Toolbox → search Hillshade (GDAL) → set Input DEM, Z Factor (1 = realistic, 2–3 = dramatic) → Run

10.4 Using QuickOSM

1
Vector > QuickOSM > QuickOSM
2
Key = amenity, Value = hospital → set extent to Layer Extent
3
Click Run Query — downloads live OSM data into QGIS
